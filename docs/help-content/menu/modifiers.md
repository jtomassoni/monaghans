# Adding Modifiers to Menu Items

Let customers customize items: "Extra Cheese", "No Onions", "Add Bacon".

## What are Modifiers?

Customization options. Examples:
- "Extra Cheese" (+$1.50)
- "No Onions"
- "Add Bacon" (+$2.00)
- "Gluten-Free Bun" (+$0.50)
- "Well Done"

## Add Modifiers

1. Go to **Menu** â†’ **Items**
2. Create new item or edit existing
3. Find **"Modifiers"** field
4. Enter modifiers separated by commas:
   ```
   Extra Cheese, No Onions, Add Bacon, Gluten-Free Bun
   ```
5. Save

## Format

Comma-separated list:
- Separate with commas
- Spaces optional
- Displayed as-is to customers

**Example**:
```
Extra Cheese, No Onions, Add Bacon, Well Done, Medium Rare
```

## Pricing

Modifiers display as text. To show pricing:
- Include in name: "Extra Cheese (+$1.50)"
- Use price notes field
- Mention in item description

**Example with Price**:
```
Extra Cheese (+$1.50), Add Bacon (+$2.00), Gluten-Free Bun (+$0.50)
```

## Common Examples

### Burgers
- Extra Cheese
- No Pickles
- Add Bacon
- Well Done
- Medium Rare

### Salads
- No Croutons
- Extra Dressing
- Add Chicken
- Gluten-Free

### Sandwiches
- Extra Mayo
- No Onions
- Add Avocado
- Toasted

### Drinks
- Extra Shot
- Decaf
- No Ice
- Extra Syrup

## Best Practices

- Keep it simple: 5-10 modifiers usually enough
- Clear names customers understand
- Consistent formatting across items
- List most common options first

## Modifiers vs. Separate Items

Create separate items when:
- Customization significantly changes item (e.g., "Veggie Burger" vs. "Burger - No Meat")
- Pricing varies significantly
- Option is completely different dish

## Display

Modifiers appear on:
- Menu item detail page
- Ordering interfaces
- Kitchen displays (if integrated)
